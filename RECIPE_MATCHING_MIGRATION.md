# Подбор рецептов через Google Apps Script

## ✨ Что изменилось?

Логика подбора рецептов **перенесена из клиента в Google Sheets** с помощью Google Apps Script:

### До:
- ❌ Клиент загружал все рецепты, продукты и инвентарь
- ❌ Клиент вычислял совпадения (медленно при большом количестве рецептов)
- ❌ Повторные вычисления при каждом открытии страницы

### После:
- ✅ Google Apps Script автоматически рассчитывает совпадения
- ✅ Результаты сохраняются в лист `RecipeMatches`
- ✅ Клиент просто **читает готовые результаты** (быстро!)
- ✅ Автоматическое обновление при изменении инвентаря

## 📦 Что было создано?

### 1. Google Apps Script (`google-apps-script/RecipeMatcher.gs`)
- Автоматически рассчитывает совпадения рецептов с инвентарем
- Обновляется при изменении листов `Inventory` или `RecipeProducts`
- Записывает результаты в лист `RecipeMatches`

### 2. Лист `RecipeMatches` в Google Sheets
Новая структура листа:
```
A: recipe_id
B: recipe_name
C: match_percentage
D: available_ingredients_count
E: missing_ingredients_count
F: total_ingredients_count
G: can_cook
H: last_updated
```

### 3. Сервис синхронизации (`recipeMatchesSync.ts`)
- Читает результаты из листа `RecipeMatches`
- Обогащает данными из других листов (Recipes, Products, Inventory)
- Возвращает полные объекты `RecipeMatch`

### 4. Обновленный хук `useRecipeMatching`
- Больше не выполняет расчеты на клиенте
- Просто читает готовые данные из Google Sheets
- Намного быстрее!

## 🚀 Установка

### Шаг 1: Установите Apps Script

Следуйте инструкциям в [google-apps-script/README.md](./google-apps-script/README.md):

1. Откройте Google Sheets
2. Расширения → Apps Script
3. Вставьте код из `RecipeMatcher.gs`
4. Запустите `setupTriggers()` для автообновления
5. Запустите первый расчет через меню **🍳 Рецепты → 🔄 Пересчитать совпадения**

### Шаг 2: Проверьте результаты

После запуска Apps Script:
1. В таблице появится новый лист `RecipeMatches`
2. Он будет содержать список всех рецептов с процентами совпадения
3. Зеленый фон = можно приготовить, Красный = нельзя

### Шаг 3: Используйте в приложении

Обновленный код уже интегрирован:
- Страница `/recipe-search` теперь читает из `RecipeMatches`
- Все фильтры и группировки работают как раньше
- Но **намного быстрее!**

## ⚠️ ВАЖНО: Требование к данным

### Все количества должны быть в БАЗОВЫХ ЕДИНИЦАХ:

- **Вес**: граммы (г)
- **Объем**: миллилитры (мл)  
- **Штуки**: шт

Apps Script **не делает конверсию единиц**. Если в таблице будут разные единицы (кг и г), расчеты будут неверными.

**Рекомендация**: При добавлении новых данных всегда конвертируйте в базовые единицы.

## 🔄 Как работает автообновление?

Google Apps Script настроен на **автоматический пересчет** при изменении:
- Листа `Inventory` (добавили/удалили продукты)
- Листа `RecipeProducts` (изменили рецепты)

Результаты автоматически обновляются в листе `RecipeMatches`.

## 🐛 Решение проблем

### Лист RecipeMatches пустой
1. Откройте Google Sheets
2. Меню **🍳 Рецепты → 🔄 Пересчитать совпадения**
3. Дождитесь завершения

### Результаты не обновляются автоматически
1. **Расширения → Apps Script**
2. Запустите `setupTriggers()` еще раз
3. Проверьте **Триггеры** (слева) - должен быть триггер `onInventoryChange`

### Неверные проценты совпадения
Проверьте, что все данные в **базовых единицах**:
- Inventory: г, мл, шт
- RecipeProducts: г, мл, шт

### Как посмотреть логи Apps Script?
1. **Расширения → Apps Script**
2. **Выполнение** (слева)
3. Просмотрите историю выполнений
