# Миграция на новую структуру данных

## Что изменилось

Главная страница Инвентаря теперь использует новую структуру данных, как описано в `sheetsManager.ts`.

### Старая структура (LegacyInventoryItem)
```typescript
{
  id: string;
  name: string;
  category: string;
  quantity: number;
  unit: string;
  expiryDate?: string;
  addedDate: string;
  notes?: string;
}
```

### Новая структура (InventoryItem)
```typescript
{
  inventory_id: string;
  product_id: string;  // Foreign Key → Products.product_id
  quantity: number;
  unit: string;
  expiry_date?: string;
  added_date: string;
  notes?: string;
}
```

## Ключевые изменения

1. **Справочник продуктов**: Теперь инвентарь ссылается на справочник Products через `product_id`
2. **Разделение данных**: Название и категория продукта хранятся в справочнике Products, а не в каждой записи инвентаря
3. **Нормализация**: Это позволяет избежать дублирования и легче управлять данными

## Как использовать

### 1. Установка Google Sheets
Перейдите в Настройки и настройте:
- Client ID
- Spreadsheet ID

### 2. Добавление продуктов
Перед добавлением в инвентарь, убедитесь что продукты есть в справочнике:
1. Перейдите в **"📦 Справочник"**
2. Добавьте необходимые продукты с:
   - Названием
   - Категорией
   - Единицей измерения по умолчанию
   - Типичным сроком годности (опционально)

### 3. Добавление в инвентарь
1. На главной странице начните вводить название продукта
2. Выберите продукт из выпадающего списка
3. Укажите количество
4. При необходимости уточните срок годности и примечания

### 4. Синхронизация
Используйте кнопку **"🔄 Синхронизировать все модули"** для синхронизации:
- Продуктов
- Рецептов
- Инвентаря
- Списка покупок
- Категорий

## Структура Google Sheets

Теперь приложение создает следующие листы:

- **Inventory**: инвентарь холодильника
- **Products**: справочник продуктов
- **Recipes**: рецепты
- **RecipeProducts**: связь рецептов и продуктов
- **ShoppingList**: список покупок
- **Categories**: категории продуктов и рецептов

## Преимущества новой структуры

1. **Консистентность**: Все продукты имеют единое написание
2. **Автозаполнение**: При выборе продукта автоматически подставляются unit и срок годности
3. **Аналитика**: Можно отслеживать usage_count для каждого продукта
4. **Масштабируемость**: Легко добавлять новые свойства продуктов
5. **Интеграция**: Единый справочник для инвентаря, рецептов и списка покупок
