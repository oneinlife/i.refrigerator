# 🔧 Устранение проблем с авторизацией Google

## Проблема: Cannot read properties of undefined (reading 'client')

### Симптомы
```
Ошибка синхронизации: Cannot read properties of undefined (reading 'client')
TypeError: Cannot read properties of undefined (reading 'client')
Google API not initialized. Please refresh the page.
```

### Причины

Это означает, что Google API не успел загрузиться или не был инициализирован.

### Решения

#### 1. Перезагрузите страницу
Самое простое и эффективное решение:
1. Нажмите **F5** или **Ctrl+R**
2. Подождите 2-3 секунды пока загрузятся скрипты Google
3. Попробуйте снова выполнить синхронизацию

#### 2. Очистите кеш браузера
1. Нажмите **Ctrl+Shift+Del**
2. Выберите "Кэшированные изображения и файлы"
3. Нажмите "Удалить данные"
4. Перезагрузите страницу

#### 3. Проверьте интернет соединение
1. Убедитесь, что интернет работает
2. Попробуйте открыть https://accounts.google.com
3. Если нет доступа - проверьте сеть

#### 4. Проверьте блокировщики рекламы
Некоторые блокировщики могут блокировать Google API:
1. Отключите AdBlock/uBlock для localhost:3000
2. Добавьте сайт в белый список
3. Перезагрузите страницу

#### 5. Проверьте консоль браузера (F12)
1. Откройте консоль разработчика (**F12**)
2. Вкладка **Console**
3. Ищите ошибки загрузки скриптов:
   - `Failed to load resource: apis.google.com`
   - `Failed to load resource: accounts.google.com`
   - `net::ERR_BLOCKED_BY_CLIENT`
4. Если есть - проблема с сетью, брандмауэром или блокировщиком

#### 6. Попробуйте другой браузер
Если ничего не помогает:
1. Откройте приложение в Chrome/Edge/Firefox
2. Проверьте работает ли там

### Техническая информация

Эта ошибка возникает когда:
- Скрипты Google API еще загружаются (слишком раннее действие)
- Скрипты заблокированы сетью/брандмауэром/блокировщиком рекламы
- Проблемы с CORS при локальной разработке
- Отсутствует интернет соединение
- Скрипты не загрузились из-за ошибки сети

### Предотвращение

**Всегда дожидайтесь полной загрузки:**
1. После открытия страницы подождите 2-3 секунды
2. Убедитесь что кнопка "Войти через Google" активна
3. Выполняйте действия только после успешной авторизации
4. Не начинайте синхронизацию сразу после авторизации - подождите 1-2 секунды

## Проблема: Браузер блокирует всплывающее окно

### Симптомы
```
[GSI_LOGGER]: Failed to open popup window...
Maybe blocked by the browser?
```

### Решение

#### Chrome / Edge
1. Посмотрите на адресную строку справа - появится иконка 🚫
2. Нажмите на нее
3. Выберите "Всегда разрешать всплывающие окна для этого сайта"
4. Нажмите "Готово"
5. Попробуйте снова "Войти через Google"

#### Firefox
1. Адресная строка → иконка 🚫 (справа)
2. "Настройки блокировки содержимого"
3. Снимите галочку "Блокировать всплывающие окна"
4. Или добавьте исключение для localhost:3000

#### Safari
1. Safari → Настройки → Веб-сайты → Всплывающие окна
2. Для localhost:3000 выберите "Разрешить"

## Проблема: Ошибка 403: access_denied

### Симптомы
```
Ошибка 403: access_denied
Параметры: client_id=..., scope=https://www.googleapis.com/auth/spreadsheets
```

### Причины и решения

#### 1. Не добавлен Authorized JavaScript Origin

**Самая частая причина!**

1. Откройте [Google Cloud Console](https://console.cloud.google.com/)
2. Перейдите в **APIs & Services → Credentials**
3. Найдите ваш **OAuth 2.0 Client ID** и откройте его
4. В разделе **"Authorized JavaScript origins"** добавьте:
   ```
   http://localhost:3000
   ```
5. Для production также добавьте:
   ```
   https://ваш-домен.com
   ```
6. Нажмите **Save**
7. Подождите 5 минут и попробуйте снова

#### 2. Приложение не опубликовано (Testing mode)

Если ваше приложение в **Testing** режиме:

1. [Google Cloud Console](https://console.cloud.google.com/) → APIs & Services → **OAuth consent screen**
2. Прокрутите вниз до секции **"Test users"**
3. Нажмите **"+ ADD USERS"**
4. Добавьте email адрес, с которым пытаетесь войти
5. Нажмите **Save**
6. Попробуйте авторизоваться снова

**Альтернатива:** Опубликуйте приложение (Publish app), но это требует верификации Google.

#### 3. Google Sheets API не включен

1. [Google Cloud Console](https://console.cloud.google.com/) → APIs & Services → **Library**
2. Найдите **"Google Sheets API"**
3. Убедитесь что статус **"Enabled"** (включено)
4. Если нет - нажмите **Enable**

#### 4. Неправильный Client ID

1. В приложении откройте **⚙️ Настройки**
2. Проверьте, что Client ID скопирован полностью
3. Формат должен быть: `*.apps.googleusercontent.com`
4. Никаких пробелов в начале/конце

### Быстрая проверка настроек

✅ **Чек-лист Google Cloud Console:**

- [ ] Google Sheets API включен (Enabled)
- [ ] OAuth 2.0 Client ID создан (Web application)
- [ ] Authorized JavaScript origins: `http://localhost:3000`
- [ ] OAuth consent screen заполнен (External, название приложения)
- [ ] Test users: добавлен ваш email (если Publishing status = Testing)
- [ ] Scopes: `https://www.googleapis.com/auth/spreadsheets` (должен добавиться автоматически)

## Проблема: Авторизация прошла, но синхронизация не работает

### Решение 1: Проверьте Spreadsheet ID
1. ⚙️ Настройки
2. Проверьте, что Spreadsheet ID заполнен
3. ID должен быть из URL таблицы:
   ```
   https://docs.google.com/spreadsheets/d/[ЭТО_ВАША_ID]/edit
   ```

### Решение 2: Проверьте Google Sheets API
1. [Google Cloud Console](https://console.cloud.google.com/)
2. APIs & Services → Dashboard
3. Убедитесь, что **Google Sheets API** включен (Enabled)

### Решение 3: Проверьте доступ к таблице
1. Откройте вашу Google таблицу
2. Убедитесь, что вы вошли под тем же аккаунтом
3. Таблица должна быть доступна для чтения/записи

## Проблема: Ошибка 403 (Permission denied)

### Причина
У вашего аккаунта нет доступа к таблице

### Решение
1. Откройте таблицу в Google Sheets
2. Проверьте, что вы вошли под правильным аккаунтом
3. Если таблица не ваша - попросите владельца дать доступ

## Проблема: Ошибка CORS

### Причина
Google OAuth не разрешает запросы с вашего домена

### Решение
1. [Google Cloud Console](https://console.cloud.google.com/)
2. APIs & Services → Credentials
3. Найдите ваш OAuth Client ID
4. Нажмите на него для редактирования
5. **Authorized JavaScript origins** должен содержать:
   - `http://localhost:3000` (для dev)
   - `https://yourdomain.com` (для production)
6. Сохраните
7. Подождите несколько минут (может занять до 5 минут)
8. Попробуйте снова

## Проблема: После авторизации появляется ошибка

### Решение: Очистка токена
1. Откройте DevTools (F12)
2. Application → Local Storage → http://localhost:3000
3. Удалите `google_access_token`
4. Перезагрузите страницу
5. Попробуйте снова войти

Или используйте кнопку **"Выйти"** в приложении.

## Проблема: "Failed to fetch" или сетевые ошибки

### Проверьте:
1. ✅ Интернет соединение работает
2. ✅ Файрвол не блокирует googleapis.com
3. ✅ VPN/прокси не мешает подключению
4. ✅ Браузер не в режиме "Работа оффлайн"

### Решение:
1. Отключите VPN (если используется)
2. Проверьте настройки файрвола
3. Попробуйте другой браузер
4. Попробуйте другую сеть (например, мобильный интернет)

## Дополнительная диагностика

### Консоль браузера (F12)
Посмотрите на ошибки в консоли. Полезная информация:
- Красные ошибки - критичные проблемы
- Желтые предупреждения - можно игнорировать
- Сообщения `[GSI_LOGGER]` - от Google Identity Services

### localStorage
Проверьте сохраненные данные:
1. F12 → Application → Local Storage → localhost:3000
2. Должны быть ключи:
   - `google_client_id`
   - `google_spreadsheet_id`
   - `google_access_token` (после авторизации)

### Сеть (Network)
1. F12 → Network
2. Попробуйте авторизоваться
3. Посмотрите на запросы к googleapis.com
4. Статус должен быть 200 (ОК)

## Все еще не работает?

```bash
cd c:\projects\i.refrigerator

# Удалить node_modules
Remove-Item -Recurse -Force node_modules

npm run dev
```

```bash

```
