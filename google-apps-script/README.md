# Google Apps Script: Recipe Matcher

Автоматический расчет совпадений рецептов с инвентарем холодильника.

## 📋 Что делает скрипт?

- Автоматически рассчитывает, какие рецепты можно приготовить из имеющихся продуктов
- Обновляет лист `RecipeMatches` при изменении инвентаря или рецептов
- Показывает процент совпадения для каждого рецепта
- Определяет, можно ли приготовить рецепт (все обязательные ингредиенты есть)

## 🚀 Установка

### Шаг 1: Откройте Apps Script редактор

1. Откройте вашу Google Таблицу
2. Перейдите в меню **Расширения** → **Apps Script**
3. Откроется редактор кода

### Шаг 2: Добавьте код

1. Удалите весь существующий код в редакторе
2. Скопируйте содержимое файла `RecipeMatcher.gs`
3. Вставьте код в редактор
4. Нажмите **💾 Сохранить** (Ctrl+S / Cmd+S)
5. Введите имя проекта: `RecipeMatcher`

### Шаг 3: Настройте триггеры (автообновление)

1. В редакторе Apps Script выберите функцию `setupTriggers` из выпадающего списка
2. Нажмите **▶️ Выполнить**
3. При первом запуске появится запрос разрешений:
   - Нажмите **Проверить разрешения**
   - Выберите ваш Google аккаунт
   - Нажмите **Дополнительно** → **Перейти на страницу "RecipeMatcher" (небезопасно)**
   - Нажмите **Разрешить**
4. Дождитесь завершения (в логах появится "✅ Триггеры настроены!")

### Шаг 4: Первый запуск

1. Вернитесь в Google Таблицу
2. Обновите страницу (F5)
3. Появится новое меню **🍳 Рецепты**
4. Выберите **🔄 Пересчитать совпадения**
5. Дождитесь завершения

## 📊 Результат

В таблице появится новый лист **RecipeMatches** со следующими колонками:

| Колонка | Описание |
|---------|----------|
| recipe_id | ID рецепта |
| recipe_name | Название рецепта |
| match_percentage | Процент совпадения (0-100%) |
| available_ingredients_count | Количество имеющихся ингредиентов |
| missing_ingredients_count | Количество недостающих ингредиентов |
| total_ingredients_count | Общее количество ингредиентов |
| can_cook | Можно ли приготовить (TRUE/FALSE) |
| last_updated | Дата последнего обновления |

## ⚙️ Автоматическое обновление

После настройки триггеров лист `RecipeMatches` будет автоматически обновляться при:
- Изменении данных в листе `Inventory`
- Изменении данных в листе `RecipeProducts`

## 🔧 Ручное обновление

Если нужно обновить данные вручную:
1. Откройте меню **🍳 Рецепты**
2. Выберите **🔄 Пересчитать совпадения**

## ⚠️ ВАЖНО: Базовые единицы измерения

Для корректной работы скрипта все количества должны быть в **базовых единицах**:

- **Вес**: граммы (г)
- **Объем**: миллилитры (мл)
- **Штуки**: шт

Скрипт **НЕ** делает конверсию единиц. Убедитесь, что:
- В `Inventory` количество указано в г/мл/шт
- В `RecipeProducts` количество указано в г/мл/шт

## 🐛 Отладка

### Просмотр логов

1. Откройте **Расширения** → **Apps Script**
2. Выберите **Выполнение** (слева)
3. Просмотрите историю выполнений и ошибки

### Ручной запуск для отладки

1. В редакторе Apps Script выберите функцию `calculateRecipeMatches`
2. Нажмите **▶️ Выполнить**
3. Проверьте логи (кнопка **Журнал выполнения** внизу)

## 📝 Примечания

- Скрипт сортирует рецепты по проценту совпадения (от большего к меньшему)
- Зеленый фон — рецепт можно приготовить
- Красный фон — рецепт нельзя приготовить (не хватает обязательных ингредиентов)
- Процент совпадения учитывает ВСЕ ингредиенты (включая опциональные)
- `can_cook` учитывает только обязательные ингредиенты
